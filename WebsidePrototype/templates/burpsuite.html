Burp Suite Free Tutorial

Einführung


Burp Suite ist eine integrierte Plattform um Sicherheitstests von Webapplikationen durchzuführen. Die darin enthaltenen Tools arbeiten gut zusammen, um einen ganzheitlichen Test vom anfänglichen Mappen und der Analyse der Angriffsoberfläche bis zum Finden und Ausnutzen von Sicherheitslücken zu ermöglichen.
Dabei bietet es zahlreiche Möglichkeiten der Automatisierung, die jedoch in der kostenlosen Version stark begrenzt sind.
Burp arbeitet als Proxy auf dem eigenen Rechner und muss als solcher zunächst im Browser eingerichtet werden. Wie der Browser dafür eingerichtet werden muss kann man auf der Supportseite von Portswigger nachlesen, hierbei sollte auch gleich das CA-Zertifikat für HTTPS-Traffic installiert werden:

https://support.portswigger.net/customer/en/portal/articles/1783055-configuring-your-browser-to-work-with-burp

Auf der VM ist Burp bereits eingerichtet. Um es zu verwenden muss zunächst die Burp Suite gestartet werden. Es befindet sich im Ordner opt und lässt sich mit den Parametern java -jar burpsuite_free_v1.6.32.jar ausführen. Anschließend wird in Iceweasle über das Proxy Managment Tool Foxy Proxy (ein bereits vorinstalliertes Addon in der Browserleiste) die Burp Suite als Proxy ausgewählt.

Funktionen


Die kostenlose Funktion bietet einige Einschränkungen im Funktionsumfang, ist jedoch bereits sehr nützlich für kleinere und einfache Aufgaben. In diesem Tutorial kann leider nicht auf sämtliche Funktionen der Burp Suite eingegangen werden. Folgende Funktionen sind in der kostenlosen Version enthalten.

Burp Proxy

Ein abfangender Proxy, der es ermöglicht Traffic zwischen dem eigenen Browser und der Zielapplikation zu inspizieren und zu modifizieren.

Verwendung:
Der Proxy ist in der Freien Version das vermutlich hilfreichste und am häufigsten verwendete Werkzeug.
Solange im Tab Intercept ein Button mit dem Namen „intercept is on“ zu sehen ist, wird jede HTTP und HTTPS Anfrage im Proxy gehalten, kann in dem Fenster darunter verändert und mittels Forward modifiziert an den Empfänger geschickt werden. Mittels Drop wird die Anfrage verworfen und über Action oder Rechtsklick lässt sich die Nachricht an ein anderes Werkzeug wie den Repeater senden und dort weiter bearbeiten.
Über die beiden History Tabs lassen sich ältere Request erneut aufrufen und verändern.
Die Options wurden eventuell bereits bei der Einrichtung benötigt. Hier lassen sich neben dem Listener auch verschiede Regeln erstellen, welche Requests abgefangen oder weitergeleitet werden sollen. In den meisten Fällen wird dies jedoch nicht benötigt.

Burp Proxy markiert bereits selbständig interessante Felder durch fette, blaue Schrifft.

Aufgabe:
Gehen sie auf Wikipedia und versuchen sie eine Wikipedia Anmeldung über Burpproxy abzufangen und zu ändern.

Burp Spider

Ein anwendungsbezogenes Spider Programm, das es ermöglicht Inhalt und Funktionalitäten eines Ziels zu durchwandern und aufzulisten. So können Beispielsweise sämtliche Adressen eines Webservers erfasst werden.

Verwendung:
Zunächst wird in den Options definiert wie und wonach gesucht wird. Anschließen kann eine Website zum Spider hinzugefügt werden. Der Spider versucht nun sämtliche Adressen des Webservers herauszufinden. Die Adressen werden im Tab Targets angezeigt. In den Grundeinstellungen wird jede Seite die besucht wird bereits mitgeloggt.

Burp Repeater

Ein Werkzeug das es ermöglicht Anfragen zu manipulieren und erneut zu senden.

Verwendung:
Zunächst wird vom Proxy eine Request an den Repeater gegeben. Im Repeater Tab kann die Anfrage nun beliebig verändert werden und mit go gesendet werden. Auf der rechten Seite wird die Rückgabe des Servers angezeigt. So kann eine Verdächtige Response manuell getestet werden.

Aufgabe:
Hohlen sie sich die zuvor manipulierte Wikipedia Anmeldung in den Repeater und versuchen sie erneut diese zu manipulieren.

Burp Intruder

Mithilfe des Intruders lassen sich leistungsstarke maßgeschneiderte Angriffe durchführen um unübliche Schwachstellen zu finden und auszunutzen.

Verwendung:
Der Intruder ermöglicht es eine Vielzahl von Eingaben auf eine Website automatisch zu fahren und die Rückgaben zu analysieren. Es wird eine Nachricht vom Proxy an den Intruder geschickt. Unter Positions werden zunächst diejenigen Elemente markiert, die verändert werden sollen. Bei einer Anmeldung Beispielsweise der Username und das Password. Unter Payload lässt sich nun definieren was in die markierten Elemente eingesetzt werden soll. Über die Options lässt sich dies noch weiter verfeinern.
In der Freien Version ist der Intruder stark beschränkt und kaum zu gebrauchen ein Testen mithilfe des Repeaters ist meist Zielführender.



Die folgenden Funktionen sind ebenfalls in der freien Version verfügbar. Werden jedoch hier nur kurz erwähnt.

Burp Sequencer

Ein Sequenzierer, für das Testen der Zufälligkeit von Session Token.

Burp Decoder

Ein einfaches Werkzeug um Codierte Daten in ihre Kanonische Form zu bringen oder Rohdaten in verschiedene codierte und gehashte Formen zu bringen. Es ist dazu in der Lage einige Codierungsformate selbständig zu erkennen.

Burp Comparer

Ein Werkzeug um die visuellen Unterschiede zweier Items miteinander vergleichen zu können.
